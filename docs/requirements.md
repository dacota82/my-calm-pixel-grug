# Requirements

## 제품 개요
**Pixel Grug** — 투박하지만 따뜻한 픽셀 동굴 친구와 함께하는 웹 앱.

톤: cozy retro RPG pixel art / 다크 + 따뜻한 earth tone.

## 핵심 사용자 흐름 (초안)
1. 앱 진입 → Grug(anchor 기준)와 첫 화면
2. 주요 행동 시작 (구체 기능은 미정)
3. 완료/빈 상태에서도 Grug 얼굴로 안내

## 기능 요구사항
| ID | 요구사항 | 상태 |
|----|----------|------|
| F-01 | 브랜드·마스코트 기준 시각 자산 보유 | 완료 (`anchor-grug.png`) |
| F-02 | 시각 정체성 문서화 | 완료 (`VISUAL_IDENTITY.md`) |
| F-03 | 앱 화면 구조(HTML/CSS/JS) | 미착수 |
| F-04 | 감정별 Grug 연출 | 예정 (승인 후) |
| F-05 | 로고·배경·소품 | 예정 (승인 후) |

## 비기능 요구사항
- **톤:** 따뜻하고 친근함. 차갑고 네온한 느낌 지양
- **가독성:** 작은 썸네일에서도 실루엣이 읽혀야 함
- **일관성:** 이후 에셋은 `assets/anchor-grug.png`를 기준으로 확장
- **문서 우선:** 생성·코딩 전에 요구사항·승인 상태를 문서에 반영

## 에셋·시각 제약
### 현재 (존재)
- `assets/anchor-grug.png` — 1차 기준 이미지

### 예정 (미생성)
- `assets/grug-mascot.png` — 앱용 마스코트 정리본
- `assets/pixel-grug-logo.png` — 로고/엠블럼
- `assets/pixel-grug-bg.png` — 동굴 배경
- 감정별 캐릭터, 소품, 버튼 상태
