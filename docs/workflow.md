# Workflow

## 일일 출정 흐름
1. 미션·요구사항 확인 (`docs/mission.md`, `docs/requirements.md`)
2. 오늘 할 일 / 하지 않을 일 구분
3. 문서 갱신 → (필요 시) 에셋·코드 작업
4. 주군에게 보고 → 승인 게이트 통과 여부 확인
5. 승인된 범위만 Git 기록

## 에셋 승인 게이트
| 게이트 | 규칙 |
|--------|------|
| Anchor | **승인 전** 추가 이미지 생성 금지 |
| 감정별 이미지 | **주군 승인 전** 생성 금지 |
| 로고·배경·소품 | **주군 승인 전** 생성 금지 |
| 기준 확장 | 항상 `assets/anchor-grug.png`를 기준으로 일관성 유지 |

현재: Anchor **승인 완료**. 감정·로고·배경·소품은 아직 생성하지 않음.

## 문서 먼저 / 생성은 승인 후
1. README·VISUAL_IDENTITY·docs와 **실제 파일 상태**를 먼저 맞춤
2. 없는 파일은 “예정”으로 분리 — 있는 것처럼 쓰지 않음
3. GenerateImage는 **승인된 1장(또는 명시된 범위)**만
4. 생성 후 경로·존재 여부 보고 → 문서 반영은 승인 후

## Git 커밋·push 규칙
- 의미 있는 단위로 커밋 (메시지에 why 중심)
- **push 전 주군 승인 필요** — 승인 없이 push 금지
- 비밀·자격 증명 파일은 커밋하지 않음

## 체크리스트
- [ ] 오늘 범위가 mission/requirements와 맞는가?
- [ ] 없는 에셋을 있는 것처럼 문서에 쓰지 않았는가?
- [ ] 이미지 생성이 승인 범위 안인가?
- [ ] 커밋 전 `git status`로 파일 목록을 확인했는가?
- [ ] push 승인을 받았는가?
